โครงสร้าง Metadata บนสุดของทุกไฟล์

Metadata เป็นส่วนที่ Orchestrator จะอ่านเพื่อตัดสินใจ Route และ Retrieval Node จะดึง doc_link มาแสดงในคำตอบ

markdown — YAML Frontmatter (บังคับทุกไฟล์) 
---
doc_id:          "REG-PDPA-001"            # รหัสเอกสารไม่ซ้ำกัน
doc_title:       "ระเบียบการคุ้มครองข้อมูลส่วนบุคคล กฟภ. 2566"
doc_type:        "ระเบียบ"                  # ระเบียบ|กฎกระทรวง|ประกาศ|คำสั่ง|แนวปฏิบัติ
domain:          "KB_IT_CYBER"
sub_domain:      "PDPA"
effective_date:  "2023-06-01"
doc_link:        "https://drive.google.com/file/d/FILE_ID/view"
source:          "กฟภ."
version:         "1.0"
tags:            ["PDPA", "ข้อมูลส่วนบุคคล", "เจ้าของข้อมูล", "DPO"]
cross_refs:
  - doc_id:    "GUIDE-PDPA-001"
    doc_title: "แนวปฏิบัติการคุ้มครองข้อมูลส่วนบุคคล"
    doc_link:  "https://drive.google.com/file/d/GUIDE_ID/view"
    reason:    "รายละเอียดปฏิบัติของระเบียบฉบับนี้"
    kb:        "KB_COMPLIANCE"           # ระบุ KB ที่เอกสารนั้นอยู่ (ถ้าคนละ KB)
---
💡 เหตุผลที่ต้องระบุ kb ใน cross_refs
เมื่อ Orchestrator อ่าน Metadata เจอ cross_refs[].kb = "KB_COMPLIANCE" จะรู้ทันทีว่าต้องส่ง Query ไปยังอีก KB ด้วย ทำให้ระบบ Cross-KB Retrieval ทำงานได้อัตโนมัติ

Format ต่างประเภท

การจัดการเอกสารแต่ละรูปแบบ

ประเภทเอกสาร วิธีจัดการ ข้อควรระวัง
PDF สแกน OCR ก่อน → ตรวจสอบเลขข้อ → แปลง ตัวเลข OCR ผิดพลาดบ่อย โดยเฉพาะ ๑, ๒, ๓ ไทย
Word (Heading) Map Heading 1→#, H2→##, H3→### ตรวจ Heading ที่ใช้ Bold แทน Heading Style
ประกาศไม่มีหมวด ใช้ ## ข้อ X เป็น Section หลัก เพิ่ม doc_type: "ประกาศ" ใน Metadata
กฎกระทรวง เพิ่ม section_type: "กฎกระทรวง" ใน YAML มักมี "มาตรา" แทน "ข้อ" — ปรับ Regex Splitter
ตารางอ้างอิง แปลงเป็น Markdown Table ทุกตาราง Merged cell ต้องแยกเป็นแถวแทน

## Task: สร้าง Cross-Reference Markdown

คุณได้รับเอกสาร 2 ฉบับ:
- เอกสาร A (ต้นทาง): {{doc_a_content}}
  doc_id={{doc_a_id}}  link={{doc_a_link}}  kb={{doc_a_kb}}
- เอกสาร B (ปลายทาง): {{doc_b_content}}
  doc_id={{doc_b_id}}  link={{doc_b_link}}  kb={{doc_b_kb}}

## งาน
1. วิเคราะห์หาความเชื่อมโยงระหว่างข้อต่างๆ
2. เพิ่ม Cross-Reference Block ทันทีหลังเนื้อหาข้อที่เชื่อมกัน:

> **📎 ดูแนวปฏิบัติเพิ่มเติม:**
> [ชื่อเอกสาร B ข้อ XX]({{doc_b_link}})
>
> <!-- XREF: doc_id={{doc_b_id}} kb={{doc_b_kb}} articles=XX,YY -->

3. เพิ่ม cross_refs ใน YAML Frontmatter ของเอกสาร A (ระบุ kb ถ้าต่าง KB)
4. เพิ่ม parent_ref ใน YAML Frontmatter ของเอกสาร B

## กฎ
- ระบุเฉพาะข้อที่เชื่อมกันจริงเท่านั้น
- ถ้าข้อใน A เชื่อมหลายข้อใน B ให้ระบุช่วง: articles=16,17,18,19,20
- ส่วนที่มองเห็น: ภาษาไทย · HTML Comment: ภาษาอังกฤษเท่านั้น
