🔍 Retrieval-Augmented Generation
RAG คืออะไร?
ความรู้พื้นฐานและแนวคิดหลักของ RAG
📚 ความรู้พื้นฐาน
RAG: การนำข้อมูลภายนอกมาเสริมพลังให้ LLM
RAG (Retrieval-Augmented Generation) คือเทคนิคที่ผสมผสาน Information Retrieval เข้ากับ LLM ทำให้โมเดลสามารถตอบคำถามด้วยข้อมูลที่เป็นปัจจุบันและแม่นยำ โดยไม่ต้อง Fine-tune โมเดลใหม่ทั้งหมด
Vector Search Embedding Knowledge Base
01
Document Chunking
การแบ่งเอกสารเป็นชิ้นเล็กๆ เพื่อให้ Embedding และ Retrieval มีประสิทธิภาพ
02
Embedding Model
การแปลงข้อความเป็น Vector ใน High-dimensional Space เพื่อ Semantic Search
03
Retrieval Strategy
กลยุทธ์ดึงข้อมูลที่เกี่ยวข้อง — Dense, Sparse, Hybrid Retrieval
04
Reranking
การจัดเรียงผลลัพธ์ใหม่ด้วย Cross-Encoder เพื่อความแม่นยำสูงขึ้น
05
HyDE
Hypothetical Document Embeddings — ให้ LLM สร้างเอกสารสมมติก่อน Retrieve
06
Context Window
การจัดการ Context ที่ส่งให้ LLM — Packing, Compression, Lost in the Middle

📋 รายละเอียดแต่ละขั้นตอน
Step 1–2
Document Loading & Chunking
รองรับไฟล์ PDF, DOCX, TXT, HTML, MD — แบ่ง Chunk ด้วยกลยุทธ์ Fixed Size (512–1024 tokens), Recursive Split หรือ Semantic Chunking ควรมี Overlap 50–100 tokens เพื่อไม่ให้ข้อมูลขาดหาย
Step 3–4
Embedding & Vector Store
แปลงแต่ละ Chunk เป็น Dense Vector ด้วย Embedding Model (เช่น bge-m3, text-embedding-3) จากนั้น Index ลงใน Vector Database เช่น Qdrant, ChromaDB, pgvector เพื่อรองรับ Similarity Search
Step 5–6
Query & Retrieval
Embed Query ด้วยโมเดลเดียวกัน ค้นหา Chunk ที่ใกล้เคียงที่สุดด้วย Cosine Similarity หรือ ANN Search ดึง Top-K (เช่น K=5–10) แล้วผ่าน Reranker เพื่อจัดลำดับใหม่
Step 7–8
Augmentation & Generation
นำ Chunk ที่ดึงมาประกอบเป็น Context ส่ง Prompt ที่มี [Context] + [Question] ให้ LLM สร้างคำตอบ ควรกำชับให้โมเดลอ้างอิงจาก Context เท่านั้น หากไม่พบให้บอกว่าไม่ทราบ

🚀 Advanced RAG
เทคนิค RAG ขั้นสูง
HyDE, GraphRAG, RAPTOR และเทคนิคล่าสุด
HyDE
Hypothetical Document Embeddings
ให้ LLM สร้างเอกสาร "สมมติ" ที่น่าจะตอบคำถาม แล้วใช้เอกสารนั้น Embed เพื่อ Retrieve แทน Query โดยตรง — ลด Vocabulary Mismatch
📄 Gao et al. 2022
GraphRAG
Graph-based RAG
Microsoft Research — ใช้ Knowledge Graph แทน Vector Search เพื่อดึงข้อมูลที่มีความสัมพันธ์เชื่อมโยงกันได้ดีขึ้น
📄 Microsoft 2024
RAPTOR
Recursive Abstractive Processing
สร้าง Hierarchical Summary Tree จากเอกสาร ช่วยให้ Retrieval ทำงานได้ดีทั้งในระดับ Detail และ Abstract
📄 Stanford 2024
Self-RAG
Self-Reflective Retrieval
LLM ตัดสินใจเองว่าต้อง Retrieve หรือไม่ และประเมินคุณภาพผลลัพธ์ที่ได้รับ
📄 Asai et al. 2023
ColBERT
Late Interaction Retrieval
เก็บ Embedding รายโทเค็น และคำนวณ Similarity แบบ Late Interaction ได้ผลดีกว่า Dense Retrieval
📄 Stanford 2021
Contextual Retrieval
Anthropic Contextual Retrieval
เพิ่ม Context ให้แต่ละ Chunk ก่อน Embedding ด้วย LLM ช่วยให้ Chunk มี Context ครบถ้วน
📄 Anthropic 2024

📊 Fundamental Analysis
วิเคราะห์ปัจจัยพื้นฐาน
เข้าใจค่าที่ใช้ประเมินมูลค่าที่แท้จริงของหุ้น
📐 Valuation Ratios
P/E
Price-to-Earnings Ratio
ราคา ÷ กำไรต่อหุ้น — ค่าต่ำ = หุ้นถูก (เชิงเปรียบเทียบในอุตสาหกรรมเดียวกัน)
P/B
Price-to-Book Ratio
เปรียบราคาตลาดกับมูลค่าทางบัญชี — <1 หมายถึงต่ำกว่าทรัพย์สิน
ROE
Return on Equity
ผลตอบแทนต่อส่วนทุน — สูงแสดงถึงประสิทธิภาพการทำกำไร ควร >15%
D/E
Debt-to-Equity Ratio
อัตราส่วนหนี้ต่อทุน — ยิ่งสูงยิ่งเสี่ยง ควรต่ำกว่า 1 ยกเว้นธนาคาร
FCF
Free Cash Flow
กระแสเงินสดอิสระหลังหักการลงทุน (CapEx) — FCF บวกและเติบโตคือสัญญาณดี
EPS
Earnings Per Share
กำไรสุทธิ ÷ จำนวนหุ้น — ดูแนวโน้มการเติบโต YoY เทียบกับ Consensus
Div
Dividend Yield
เงินปันผลต่อปี ÷ ราคาหุ้น — Yield 3–6% ถือว่าน่าสนใจสำหรับนักลงทุนระยะยาว
EV
EV/EBITDA
Enterprise Value ÷ กำไรก่อนหักดอกเบี้ยและภาษี — ใช้เปรียบเทียบข้ามบริษัทที่มีหนี้ต่างกัน
📋 Framework วิเคราะห์หุ้น
Moat
Economic Moat (ความได้เปรียบเชิงแข่งขัน)
คำถามสำคัญ: บริษัทนี้มีอะไรที่คู่แข่งลอกเลียนได้ยาก? เช่น Network Effect (LINE), ต้นทุนสลับ (Oracle), Brand (Apple), Patent, Scale Advantage
DCF
Discounted Cash Flow

⚡ Node Types ใน Workflow App
Start
Start Node
รับ Input จากผู้ใช้ — ใช้ {{#start.query#}} ใน Workflow (ต่างจาก Chatflow ที่ใช้ sys.query)

LLM Node

If/Else
Condition Node

Code
Code Node (Python/JS)

Iter
Iteration Node

⚠️ ข้อควรระวัง (Gotchas)
Code Node
Variable ต้องเป็น ASCII เท่านั้น
Code Node ใน Dify รับได้เฉพาะชื่อ Variable แบบ ASCII — ห้ามใช้ภาษาไทย, em-dash (—), หรือ Backtick ใน variable name มิฉะนั้นจะ Error
Iteration
Input ต้องเป็น Array Type
Iteration Node รับเฉพาะ Array — หาก LLM Output เป็น String ต้องผ่าน Code Node แปลงด้วย json.loads() ก่อน
Workflow Query
ตัวแปร Query ใน Workflow App
ใน Workflow App ใช้ {{#start.query#}} — ไม่ใช่ {{sys.query}} แบบ Chatflow เพราะจะ Error
