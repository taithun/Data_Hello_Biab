What Hello Biab ทำได้ — 3 ระดับ Benefit

LEVEL 1 — ความเร็ว
ประหยัดเวลา 18 นาที/session

LEVEL 2 — ความครบถ้วน
ตอบได้ครบ ไม่พลาดข้อที่เกี่ยวข้อง
ระบบเดิม "หาเจอแต่ไม่ครบ" เพราะ Ctrl+F หาได้แค่คำที่ค้น — HyDE+Rerank ดึง Chunks ที่เกี่ยวข้องทั้งหมดมาให้ ลด Compliance Risk
LEVEL 3 — Unmet Demand
ตอบ 60% ที่ระบบเดิมตอบไม่ได้
~29 คน/วันที่หาไฟล์ไม่เจอ คือ Demand ที่รอการตอบสนองอยู่แล้ว ไม่ต้องสร้างใหม่ — Adoption Rate 2x จึง Conservative มาก

token usage & api cost breakdown

ราคาอ้างอิง: Claude Sonnet $3/M input tokens, $15/M output tokens · Rerank model $2/1,000 queries · Embedding (Large) $0.13/M tokens · อัตราแลกเปลี่ยน USD/THB = 37

1
HyDE Generation

~300 tokens | ค่า Embedding ถูกกว่า LLM มาก

สูตรคำนวณ B/C Ratio และ ROI ฉบับเต็ม

// === ฝั่ง BENEFIT — คงที่ตาม Sessions ไม่ใช่ Queries ===
cost_per_min = salary/เดือน ÷ (วันทำงาน/เดือน × hrs_per_day × 60)
             = 35,000 ÷ (20 × 8 × 60) = 3.646 ฿/นาที

benefit_per_session = (manual_min - ai_min) × cost_per_min
                    = (22 - 4) × 3.646 = 65.63 ฿/session

annual_benefit = sessions/วัน × work_days × benefit_per_session
// หมายเหตุ: 15-30 นาที = ทั้งกระบวนการรวม Follow-up แล้ว → Benefit ไม่เพิ่มตาม Queries

// === ฝั่ง COST — เพิ่มตาม Queries ===
queries/ปี = sessions/วัน × work_days × queries_per_session
api_cost = queries/ปี × cost_per_query // 1.045 ฿ (P1) หรือ 1.32 ฿ (P2)
total_cost = api_cost + infra_annual

// === B/C และ ROI ===
BC_ratio = annual_benefit ÷ total_cost
ROI = (annual_benefit - total_cost) ÷ total_cost × 100 // %
payback_months = total_cost ÷ annual_benefit × 12

// ผลของ Queries/Session: Cost ↑ แต่ Benefit คงที่ → B/C ลดลง (Conservative)
// ตัวอย่าง 200 users, Phase 1, 1 q/session: BC = 3,150K ÷ 110K = 28.6x
// ตัวอย่าง 200 users, Phase 2, 5 q/session: BC = 3,150K ÷ 413K = 7.6x

ข้อจำกัดผลกระทบวิธีรับมือ
Queries/Session สมมติที่ 1 (default)ถ้าผู้ใช้ถามเฉลี่ย 3 q/session → Cost เพิ่ม 3x แต่ Benefit เท่าเดิมใช้ Slider ในหน้าภาพรวมเพื่อดู Sensitivity ก่อนนำเสนอ
Benefit = Opportunity Cost ไม่ใช่ Cashผู้บริหารอาจโต้แย้งระบุ Use Case ที่เวลาถูกนำไปใช้จริง
GA4 วัดได้แค่ขั้นค้นหาไฟล์ ไม่เห็นต้นทุนหลัง Downloadต้นทุนจริงอาจสูงกว่า Benefit ที่คำนวณโมเดลนี้ Conservative — เป็นข้อได้เปรียบเมื่อนำเสนอ
Adoption Rate 2x — อาจต่ำกว่าจริงUnmet Demand 60% (~29/วัน) รอการตอบสนองอยู่แล้ว200 คน/วัน (Realistic) = Conservative และป้องกันการโต้แย้งได้
ราคา API อาจเปลี่ยนแปลงผลกระทบน้อยเพราะ API = 45% ของ costMonitor ทุกไตรมาส
Cross-KB Query ประมาณ 20%ถ้าสูงกว่านี้ Phase 2 cost เพิ่มออกแบบ Orchestrator รองรับ Multi-Retrieval
